## Reporting a Vulnerability

We take the security of our project seriously and appreciate the community's efforts to responsibly disclose vulnerabilities. If you discover a security vulnerability, we encourage you to report it to us as quickly as possible.

**Please DO NOT open a public issue on GitHub or discuss the vulnerability in public forums unless explicitly instructed to do so after initial contact.**

### How to Report

To report a vulnerability, please open a new issue on our GitHub repository. Please mark the issue as "confidential" if your repository supports it, or clearly state in the title that it is a security vulnerability.

### What to Include in Your Report

To help us quickly understand and address the issue, please include as much of the following information as possible:

*   **Vulnerability Type:** (e.g., XSS, SQL Injection, RCE, Authentication Bypass, etc.)
*   **Affected Version(s):** Specify the exact version(s) of the project where the vulnerability exists.
*   **Steps to Reproduce:** Provide clear, step-by-step instructions to reproduce the vulnerability. Include any necessary code, configuration, or environment details.
*   **Proof-of-Concept (PoC):** If possible, include a minimal working example or screenshot(s) demonstrating the vulnerability.
*   **Impact:** Describe the potential impact of the vulnerability (e.g., data breach, system compromise, denial of service).
*   **Suggested Fix (Optional):** If you have a proposed solution, please include it.

### Our Commitment and Response Times

*   **Acknowledgement:** We aim to acknowledge receipt of your vulnerability report within **2-3 business days**.
*   **Investigation:** We will investigate the reported vulnerability promptly. You will receive regular updates on the status of our investigation.
*   **Resolution:** We strive to resolve critical vulnerabilities as quickly as possible. Our typical timeline for resolution depends on the severity and complexity of the issue, but we aim for a fix within **30-90 days** for critical issues.
*   **Coordinated Disclosure:** We follow a coordinated disclosure policy. Once a fix is ready, we will work with you to agree on a disclosure timeline. We typically prefer to disclose the vulnerability publicly only after a patch is widely available.
*   **Recognition:** We are grateful for responsible disclosures and will gladly credit you for your discovery, if you wish to be acknowledged, in our release notes or security advisories.

Thank you for helping to keep our project secure!
